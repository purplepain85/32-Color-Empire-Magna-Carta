# ART BIBLE v1.0  
## Section XII — Metadata & Trait Rules  
*(Trait Categories, Rarity Logic, Chain Variants, Consistency Standards)*

---

## XII.1 — Purpose of Metadata Rules
Metadata is the **technical backbone** of the Empire’s digital identity.  
It ensures:

- Consistency across all dragons  
- Predictable rarity distribution  
- Accurate trait interpretation  
- Cross‑chain compatibility  
- Future‑proof expansion  

Metadata is not decoration — it is constitutional structure.

---

## XII.2 — Core Metadata Principles
All metadata must follow these rules:

- No redundant traits  
- No contradictory traits  
- No traits that break color identity  
- No traits that break anatomy rules  
- No traits that break chain identity  
- All traits must be deterministic and reproducible  

Metadata must be clean, logical, and immutable.

---

## XII.3 — Trait Category Structure
Each Genesis Dragon uses the following trait categories:

1. **Color Identity**  
2. **Chain (Solana or Polygon)**  
3. **Symbolic Geometry**  
4. **Pose Category**  
5. **Wing Position**  
6. **Horn/Crest Variant**  
7. **Tail Variant**  
8. **Aura Type**  
9. **Background Complexity Level**  
10. **Special Effects (Optional)**  

No additional categories may be added without constitutional approval.

---

## XII.4 — Rarity Logic
Rarity must be:

- Predictable  
- Transparent  
- Mathematically balanced  
- Color‑aligned  
- Chain‑aligned  

### Rarity tiers:
- **Common:** Standard poses, standard horns, minimal backgrounds  
- **Uncommon:** Variant horns, variant wings, symbolic backgrounds  
- **Rare:** Unique geometry, enhanced aura, special material variants  
- **Mythic:** Reserved for lore‑specific dragons only  

No randomness.  
No artificial scarcity.  
Rarity must reflect **lore and identity**, not luck.

---

## XII.5 — Chain‑Specific Metadata Variants

### **Solana Metadata Style**
- Fluid descriptors  
- Organic terminology  
- Soft gradient references  
- Iridescent material tags  

### **Polygon Metadata Style**
- Structured descriptors  
- Geometric terminology  
- Crystalline material tags  
- Faceted light references  

Both must remain compatible and aligned.

---

## XII.6 — Forbidden Metadata Practices
These break the Empire’s technical law:

- Duplicate traits  
- Conflicting traits  
- Traits that contradict anatomy  
- Traits that contradict color identity  
- Traits that introduce sci‑fi or cartoon elements  
- Traits that imply rarity manipulation  
- Traits that break chain identity  

Metadata must remain pure and constitutional.

---

## XII.7 — Metadata Integration with Visual Rules
Metadata must always match:

- Pose  
- Anatomy  
- Color identity  
- Symbolic geometry  
- Background complexity  
- Aura behavior  
- Chain variant  

Metadata and visuals must never contradict each other.

---

## XII.8 — Metadata Consistency Mandate
All metadata across the Empire must follow:

- The same category structure  
- The same rarity logic  
- The same chain‑specific rules  
- The same color‑aligned identity  
- The same constitutional constraints  

Metadata is the Empire’s technical truth.  
It must remain stable across all eras, artists, and expansions.
